Getting Started

Evothings Studio consists of three connected parts, Evothings Workbench (the application you are currently running), the Evothings Viewer app running on your mobile device(s) and cloud servers to connect the Workbench and the Viewer applications. You need all three parts to be able to use Evothings Studio.

Run apps on connected Viewers by clicking on the apps listed under Examples and My Apps. The currently running app will be automatically reloaded when you edit and save any of the source code files!

Step 1: Install Evothings Viewer

Install the Evothings Viewer on your mobile device(s), get it from Google Play or Apple App Store.

Step 2: Input Cloud Token

Input the cloud token at the Connect page. You only need to do this once. The cloud token give you access to the cloud servers needed to use Evothings Studio.

Step 3: Connect to the Workbench from the Viewer

To connect the Evothings Viewer to the Workbench, you need Internet access and a connect key.

Go to the Connect screen and click the button to obtain a connect key. Start the Viewer app on your mobile device and enter this key to connect to your Workbench. Once connected, your session has no time limit. Click the "Get Key" button again if the connect key times out.

Click in the status bar below to get a Quick-Connect button in the Evothings Viewer app. This button is displayed when you restart the Viewer and allows you to reconnect to the Workbench where you are logged in, without entering a connect key again. This makes it much quicker to connect to the same Workbench.

Step 3: How to create a new app

To create your own apps you have three options. Copy one of the Evothings example apps by clicking the button, and modify it for your own purposes, or click the button to create a basic app from a template.

You can also drag and drop any .html file (typically index.html) to the Workbench window to add an existing app under My Apps.

To add a project with ECMAScript 6 support, drag the project folder or the evothings.json file into the Workbench window. Learn more in the ECMAScript documentation for Evothings Studio.

Arrange apps using drag and drop (the example apps cannot be rearranged though). Click the close icon (x) to delete an app entry. This will NOT delete the application files.

Step 4: Code in your favourite editor end enjoy live reload

Launch the app you wish to work on in connected Viewers using the button. This makes it the active app. When you save any code file for the active app, it will automatically get reloaded on all connected mobile devices.

To edit the code of an app, click the button on the app under My Apps. This will open a system file browser for your app. Note that you cannot edit the examples directly, you need to make a copy first.

From the system file browser, open the files you wish to edit in your editor of choice. Edit the file and press save in the editor to reload the app.

For further details, please visit the online documentation.

Happy coding!

Connect

Evothings Studio consists of the Evothings Workbench communicating with the Evothings Viewer mobile app through the Evothings cloud servers enabling the fast workflow and other features.

Read the Getting Started Guide for detailed first-time instructions. You get the Evothings Viewer mobile app from Google Play or Apple App Store.

To run apps in the Evothings Viewer, you need Internet access, a cloud token that you enter once to get access to our cloud infrastructure, and a connect key to connect the Viewer.

Input your cloud token below and press save. You only need to do this once. The cloud token is generated at our download page and gives you access to our cloud servers. The token is anonymous whereby you can use Evothings Studio without entering personal information.

Enter Cloud Token. Get your cloud token at the download page: https://evothings.com/download

Cloud Token:

Click the Get Key button to obtain a connect key. Use this key to connect from the Evothings Viewer app. Click "Get Key" again if the key times out. Once connected, your session has no time limit.

Click the Run buttons on the Examples screen to run an app in the Viewer.

Personalize your workbench by using the Personalize button in the status bar. This will give access to added features and services based on that the Viewer can identify a unique Workbench. Currently you get access to a convenient feature where Viewers can be reconnected to a Workbench with a one click option. Viewers and users need to be able to know which Workbench to connect to when there are several possible ones. A common scenario when teams use Evothings is that Viewers are connected to different workbenches for development and testing purposes. We plan to add features and services going forward.

Please go to the authentication window that has been opened in your system browser to complete personalization.
